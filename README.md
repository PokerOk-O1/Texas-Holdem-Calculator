# Калькулятор вероятностей в Техасском Холдеме

Однофайловый калькулятор эквити для Техасского Холдем-покера (NL Texas Hold’em), написанный на Python. Скрипт `calculator.py` оценивает шансы на победу в раздаче Техасского Холдем против одного или нескольких оппонентов.  

**Поддерживается:**
- расчёт **эквити героя**:
  - вероятность победы,
  - вероятность ничьей,
  - вероятность поражения;
- режим **Монте-Карло** (по умолчанию);
- попытка использования **точного перебора** (`--mode exact`) в простых сценариях;
- работа как с **фактическими руками оппонентов**, так и с их **диапазонами**;
- вывод в **человекочитаемом формате** или в формате **JSON**.

Скрипт предназначен для обучения, анализа розыгрышей и демонстрации алгоритмов работы с покерной комбинаторикой.

---

## 🃏 Поддерживаемые форматы карт и диапазонов

### Формат карт

Каждая карта задаётся строкой из двух символов:

- Ранги: `2 3 4 5 6 7 8 9 T J Q K A`
- Масти:
  - `c` — clubs (трефы),
  - `d` — diamonds (бубны),
  - `h` — hearts (червы),
  - `s` — spades (пики).

Примеры карт: `As`, `Kd`, `Tc`, `7h`.

Карты можно передавать:

- **без пробелов**:  
  `AsKd`  
- **через пробел**:  
  `As Kd`

Оба варианта корректно парсятся.

---

### Формат диапазонов

Диапазоны задаются в стиле покерных софтов:

- Пары:  
  - `22`, `55`,  
  - `55+` — все пары от 55 и старше (55, 66, 77, …, AA).
- Одномастные руки:  
  - `A2s`, `KTs`,  
  - `A2s+` — A2s, A3s, …, AKs.
- Разномастные:  
  - `KTo`, `QJo`,  
  - `KQo+` — KQo, KJo и т.п. (в пределах допустимого).
- Без указания масти (`s/o`):  
  - `AK` — любая комбинация AK (и одномастная, и разномастная).
  - `AQ+` — AQ, AK (в любых мастях).

Диапазон — это строка из нескольких токенов, разделённых запятыми:

```bash
"22+,A2s+,KTs+,QJo+"
````

## 🔧 Параметры командной строки

Основной интерфейс реализован через `argparse`.

### Обязательные и основные параметры

#### `--hero` / `-H`

Карманная рука героя.

* Формат: 2 карты через пробел или слитно.
* Примеры:

  * `"As Kd"`
  * `"AsKd"`

```bash
--hero "As Kd"
```

#### `--board` / `-B`

Карты борда (0–5 карт).

* Примеры:

  * `""` — пустой борд (префлоп),
  * `"2c 7d Jh"`,
  * `"2c7dJh"`.

```bash
--board "2c 7d Jh"
```

#### `--villain-hand`

Фиксированная карманная рука оппонента.

* Можно указывать несколько раз — по одному оппоненту на флаг.
* Примеры:

```bash
--villain-hand "Qd Qs" \
--villain-hand "9h 8h"
```

#### `--villain-range`

Диапазон рук оппонента.

* Можно указывать несколько раз — для разных оппонентов.
* Пример:

```bash
--villain-range "22+,A2s+,KTs+,QJo+"
```

Можно комбинировать **конкретные руки** и **диапазоны**.

---

### Режим расчёта

#### `--mode`

* `mc` — **режим Монте-Карло** (по умолчанию).
* `exact` — попытка использовать **полный перебор**.

```bash
--mode mc
# или
--mode exact
```

**Важно:** если `--mode exact` невозможен (слишком много комбинаций, диапазоны вместо фиксированных рук и т.п.), скрипт автоматически переключается на Монте-Карло и выводит соответствующее уведомление.

---

### Количество симуляций

#### `--iterations` / `-N`

Количество итераций для Монте-Карло.

* Тип: целое число.
* Значение по умолчанию: `100000`.

```bash
--iterations 200000
```

---

### Seed для генератора случайных чисел

#### `--seed`

Позволяет зафиксировать генератор случайных чисел, чтобы при одном и том же наборе параметров воспроизводить одинаковый результат.

```bash
--seed 42
```

---

### Формат вывода

#### `--json`

Если указан, результат выводится в формате JSON.

```bash
--json
```

#### `--verbose`

Расширенный вывод:

* интерпретация диапазонов,
* количество развёрнутых комбинаций,
* информация о выбранном режиме расчёта.

```bash
--verbose
```

---

### Режим самопроверки

#### `--self-test`

Запускает набор встроенных тестов (парсинг карт, базовые проверки эквити) и завершает работу.

```bash
python calculator.py --self-test
```

## 🧠 Внутреннее устройство (кратко)

Всё реализовано в одном файле `calculator.py`, но логически разделено на блоки:

1. **Модель карты и колоды**

   * Класс `Card` (ранг + масть).
   * Генерация полной колоды (52 карты).
   * Функции парсинга строк в карты.

2. **Оценка покерной руки**

   * Функция `evaluate_5cards` — определяет категорию руки и кикеры.
   * Функция `evaluate_7cards` — перебирает все 5-карточные комбинации из 7 карт.
   * Категории: от High Card до Straight Flush.

3. **Диапазоны**

   * Парсинг токенов диапазона (`55+`, `A2s+`, `KQo+`, `AK` и т.д.).
   * Разворачивание диапазона в конкретные стартовые комбинации с учётом мастей.

4. **Симуляция**

   * Режим **точного перебора** (`exact`) — при небольшом количестве неизвестных карт и игроков.
   * Режим **Монте-Карло** (`mc`) — случайный добор рук оппонентов (из диапазонов) и недостающих карт борда.
   * Подсчёт количества побед, ничьих и поражений.

5. **CLI**

   * Обработка аргументов командной строки.
   * Валидация ввода (дубликаты карт, пустые диапазоны и т.п.).
   * Формирование итогового вывода (текст/JSON).

## ✅ Валидация и обработка ошибок

Скрипт проверяет:

* корректность формата карт;
* отсутствие **дубликатов** карт между героем, бордом и руками оппонентов;
* наличие хотя бы одного оппонента;
* корректность диапазонов (некорректные токены пропускаются с предупреждением);
* достижимость режима `--mode exact` (иначе — fallback на Monte Carlo с предупреждением).

При критических ошибках входных данных скрипт завершает работу с сообщением об ошибке.

## 🧪 Режим самопроверки (`--self-test`)

Есть набор встроенных тестов, который можно запустить так:

```bash
python calculator.py --self-test
```

Проверяется:

* корректность парсинга карт;
* базовые соотношения эквити:

  * `AA` сильнее `KK` префлоп;
  * одинаковые руки почти всегда делят банк;
  * готовый натсовый флеш/стрит-флеш на полном борде побеждает более низкий флеш.

При успешном прохождении выводится:

```text
Self-test passed
```
